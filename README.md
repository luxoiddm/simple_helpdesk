# SupportDesk — Современная система Helpdesk

SupportDesk — это проcтое и легкое решение для управления заявками, поддержки клиентов и контроля выполнения задач. Система обеспечивает эффективное взаимодействие между клиентами и специалистами техподдержки в режиме реального времени.

## Основные возможности

### 1. Управление заявками (Тикетами)
- **Создание заявок**: Клиенты могут легко создавать новые обращения с описанием проблемы.
- **Статусы и приоритеты**: Каждая заявка имеет статус (`new`, `open`, `pending`, `resolved`) и приоритет (`low`, `medium`, `high`, `critical`).
- **SLA и дедлайны**: Автоматический расчет дедлайнов для контроля времени выполнения.
- **Архив и поиск**: Полноценный раздел «Заявки» с поиском по ID, теме или клиенту, а также фильтрами по статусу и приоритету.

### 2. Коммуникация в реальном времени
- **Чат**: Мгновенный обмен сообщениями между клиентом и поддержкой через WebSockets.
- **Интеграция с Email**: Автоматическое создание заявок из входящих писем (IMAP) и отправка ответов клиенту на почту (SMTP). Поддержка цепочек писем (threading) по теме и ID.
- **Внутренние заметки**: Специалисты поддержки могут оставлять скрытые комментарии, видимые только другим сотрудникам.
- **Уведомления**: Обновление статусов и новые сообщения отображаются в интерфейсе без перезагрузки страницы.

### 3. Аналитика и Дашборды
- **Визуализация данных**: Графики активности за неделю и распределение заявок по статусам.
- **Статистика**: Быстрый доступ к количеству открытых, решенных и критических заявок.

### 4. Администрирование
- **Управление пользователями**: Регистрация клиентов, сотрудников поддержки и администраторов.
- **Ролевая модель**: Разграничение прав доступа (Admin, Support, Client).
- **Сессии**: Сохранение авторизации после перезагрузки страницы (F5).

## Инструкция по установке и запуску

### 1. Требования
- **Node.js**: версия 18 или выше.
- **npm**: версия 8 или выше.
- **ОС**: Linux (Ubuntu/Debian рекомендуется), macOS или Windows.

### 2. Установка зависимостей
Перейдите в корневую директорию проекта и выполните:
```bash
npm install
```
Это установит все необходимые пакеты для фронтенда (React, Vite, Tailwind CSS) и бэкенда (Express, Socket.io, SQLite3).

### 3. Настройка окружения
Создайте файл `.env` в корне проекта на основе примера:
```bash
cp .env.example .env
```
Отредактируйте `.env` и заполните следующие параметры:
- `JWT_SECRET`: Секретный ключ для подписи токенов (установите свой).
- `API_KEY`: Ключ для доступа к внешнему API.
- `PORT`: Порт сервера (по умолчанию 3030).
- `ADMIN_USERNAME` и `ADMIN_PASSWORD`: Данные для первого входа администратора.
- **Email (IMAP/SMTP)**: Настройки вашей почты для работы тикет-системы через email.

### 4. Запуск в режиме разработки
Для одновременного запуска сервера и фронтенда с поддержкой горячей перезагрузки:
```bash
npm run dev
```
Приложение будет доступно по адресу `http://localhost:3030`.

### 5. Сборка и запуск в продакшене
1. Соберите фронтенд:
   ```bash
   npm run build
   ```
2. Запустите сервер:
   ```bash
   npm start
   ```

### 6. Запуск через PM2 (рекомендуется для серверов)
Для обеспечения бесперебойной работы приложения в фоновом режиме используйте менеджер процессов PM2:

1. Установите PM2 глобально (если еще не установлен):
   ```bash
   npm install -g pm2
   ```
2. Запустите приложение:
   ```bash
   pm2 start npm --name supportdesk -- run start
   ```
3. Настройте автозапуск при перезагрузке сервера:
   ```bash
   pm2 save
   pm2 startup
   ```

## Технологический стек

- **Frontend**: React 19, Vite, Tailwind CSS 4, Framer Motion, Recharts, Lucide-react.
- **Backend**: Node.js, Express, Socket.io, IMAPFlow, Mailparser, Nodemailer.
- **База данных**: SQLite (хранится в файле `database.db`).
- **Безопасность**: JWT-аутентификация, хеширование паролей bcryptjs.

## Использование API

SupportDesk предоставляет API для интеграции с внешними сервисами. Подробную информацию о методах и аутентификации см. в файле `api_help.md`.
